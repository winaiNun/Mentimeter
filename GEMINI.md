# Instructions for Gemini

Before doing any work in this repo, read [PROJECT_CONTEXT.md](./PROJECT_CONTEXT.md)
in full. It has the project overview, source map, hard-won conventions/gotchas,
and a dated session log of recent work — everything needed to get up to speed
without re-deriving it from the code.

When you finish meaningful work, append a dated entry to that file's
**Session log** section (and update **Open / Pending**). This is the shared
handoff doc between Claude and Gemini sessions — keep it current so whichever
tool picks up next can continue immediately.
